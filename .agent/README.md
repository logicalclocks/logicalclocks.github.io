# Agent Docs

This repo builds and publishes `https://docs.hopsworks.ai` using mkdocs and mike.
There is no application code — all work is writing Markdown under `docs/` and keeping `mkdocs.yml` in sync.

## Commands

```bash
uv venv && uv pip install -r requirements-docs.txt # setup
uv pip install "git+https://github.com/logicalclocks/hopsworks-api.git@main#subdirectory=python" # needed for Python API section
touch docs/javadoc; uv run mkdocs serve; rm docs/javadoc # preview with live reload
touch docs/javadoc; uv run mkdocs build -s; rm docs/javadoc # build in strict mode
npx markdownlint-cli2 "**/*.md" # lint Markdown (requires Node.js)
uv tool install md-snakeoil && snakeoil --line-length 88 --rules "E,F,B,C4,ISC,PIE,PYI,Q,RSE,RET,SIM,TC,I,W,D2,D3,D4,INP,UP,FA" docs # lint Python code blocks
```

`docs/javadoc` is a directory generated by CI from the `hopsworks-api` Java source.
Locally it must exist as a stub (`touch`) for the build to pass.

## Content Structure

Three directories under `docs/`, each for a distinct purpose:

- `concepts/` — explains what something is and why it works that way; no step-by-step instructions
- `user_guides/` — explains how to use a feature; task-focused, procedural
- `setup_installation/` — infrastructure setup, cloud deployment, admin configuration

Every new page must also appear in the `nav:` section of `mkdocs.yml` or it will not be reachable.

## Python API Section

The Python API docs are auto-generated from docstrings in the `hopsworks-api` repository via the `hopsworks-apigen` mkdocs plugin.
Do not write prose in API reference pages in this repo — edit the docstrings in `hopsworks-api` instead.
