The `nav:` in `mkdocs.yml` references `docs/javadoc`, which is a directory generated by CI from the `hopsworks-api` Java source.
Locally it does not exist, so `mkdocs build -s` aborts immediately with a warning.

Create an empty stub before building and remove it after: `touch docs/javadoc && uv run mkdocs build -s && rm docs/javadoc`.
This is exactly what CI does.
